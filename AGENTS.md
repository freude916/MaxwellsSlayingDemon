# **./AGENTS.md**

# 这是什么？

./AGENTS.md 是您的长期记忆，每次您被唤醒时都被加载。

如果您发现您花费了相当长的时间来解决某个可能具有通用性的问题，或者您发现自己在不断地重复同样的错误，
或者用户多次对某个效果纠正或犹豫不决（这意味着这个效果可能是有设计难度或者沟通难度的），
那么请将这些经验简要地写下来并保存在这里。

# 这个项目是什么？

这是一个杀戮尖塔2 Mod，技术栈上使用 C#, Godot, Harmony。

# 开发资源

理论上有一个 .reference 目录，里面放了一些参考项目。

另外走到 ~/Documents/MTSII/ ，你可以看到 _sts2src 和 _sts2res 两个目录，里面分别放了游戏的反编译源代码和资源文件（图片、场景等），还有一些反编译工具。

以及走到上级目录，这里有很多 Mod 项目，其中最重要的：

- BaseLib 项目，是二代的 ModTheSpire+stslib 的合体，里面有一些工具类和一些对游戏的反射封装，
- BaseLib-Wiki 是 BaseLib 的文档，里面有一些使用说明。

拿不准的可以去参考。 （注意参考前最好 ls -l 看看文件的修改时间，如果超过一周了可能就过时了，对于 Mod 项目来说你可以试试 git
pull 来更新一下）

请您优先执行 dotcheck.sh 代替 dotnet build 来构建项目，它不返回 Warning，能加快 check 速度和减少上下文浪费。

# 开发假设

所有 mod 不包含任何持久化，所以不用考虑任何数据存储和版本兼容问题，比如上限更改就直接改不要做钳制，Keep It Simple & Stupid.

不要跨层修改！ Ui 做 Ui 的活，服务做服务的活，命令做命令的活。如果抽象坏掉了，停下工作并请求 User 介入。

# 卡牌开发

## 资源文件命名

图片资源文件名需要从驼峰变成使用**下划线分隔**（snake_case），并且需要命名空间。例如：

- `HeatSource` → `maxwellmod_heat_source.png`

## 温度效果约定

策划组经常会口胡，一般来说“变热/变冷” 的实际意义是 “升温1/降温1”，除非它是 “强制变冷/强制变热”（一般不会）。

遇到“变热/变冷”请你大概像这么说一句“目前我在按照升温1/降温1来实现这个效果，如果你想要强制变热/变冷请告诉我”，然后继续按照升温1/降温1来实现。
